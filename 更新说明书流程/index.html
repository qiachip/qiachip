    更新说明书流程

    本文档描述了如何按照标准格式更新产品说明书的流程，以确保所有产品文档保持一致性和专业性。

    1. 文件命名规范

    标准命名格式

    产品说明书的 Markdown 文件应遵循以下命名规范： 

    QIACHIP_[产品型号].md

    示例

    • 正确：QIACHIP_KR1201A.md
    • 错误：QIACHIP KR2202 Instruction Manual AC 110V 220V 433 2ac821bc22e7805eb1f8c51abe323303.md

    2. 文档格式规范

    2.1 文件头部

    每个 Markdown 文件应包含以下 YAML 前置元数据： 

    ---
comments: true
---

    2.2 标题格式

    • 主标题使用一级标题（#）
    • 各章节使用二级标题（##）
    • 子章节使用三级标题（###）
    • 不要在标题中使用粗体标记（**）

    2.3 图片引用格式

    所有图片引用应包含以下属性： 

    ![图片描述](图片路径){ width="50%" .center loading="lazy" }

    • 产品图片使用 width="50%"
    • 尺寸图使用 width="68%"
    • 所有图片应居中显示并添加懒加载属性

    2.4 组件描述格式

    组件描述应使用 HTML 布局，分为两列显示： 

    <div style="display: flex; flex-wrap: wrap; justify-content: space-around;">
  <ul style="flex: 1 1 45%; margin-right: 1%;">
    <li>组件1: 描述</li>
    <li>组件2: 描述</li>
    <!-- 更多组件... -->
  </ul>
  <ul style="flex: 1 1 45%; margin-left: 1%;">
    <li>组件3: 描述</li>
    <li>组件4: 描述</li>
    <!-- 更多组件... -->
  </ul>
</div>

    2.5 表格标题

    表格标题使用小写： - 正确：## Electrical characteristics - 错误：# Electrical Characteristics

    3. 更新流程

    步骤 1：检查现有文件

    1. 确定需要更新的产品文档
    2. 检查当前文件名是否符合命名规范
    3. 检查当前文件格式是否符合标准

    步骤 2：重命名文件（如需要）

    如果文件名不符合命名规范： 1. 使用标准命名格式重命名文件 2. 确保文件名只包含产品型号，不包含额外描述

    步骤 3：更新文件格式

    按照以下顺序更新文件格式： 1. 添加 YAML 前置元数据 2. 调整标题层级（将一级标题改为二级标题等） 3. 更新图片引用，添加所需属性 4. 调整组件描述为两列布局 5. 检查并修正表格标题大小写 6. 移除标题中的粗体标记

    步骤 4：更新首页链接（如需要）

    如果产品文档的文件名或图片路径发生变化： 1. 检查 docs/index.md 中的产品链接 2. 确保链接指向正确的新文件名 3. 确保图片路径与实际文件名一致 4. 验证所有产品分类下的链接都正确

    步骤 5：验证更新

    1. 检查文件格式是否与标准文档一致
    2. 确保所有图片引用正确
    3. 验证 Markdown 语法是否正确
    4. 检查是否有遗漏的格式更新
    5. 测试首页链接是否正常工作

    4. 注意事项

    1. 保持内容不变：更新过程中不应改变文档的实际内容，只调整格式
    2. 图片路径：确保图片引用路径正确，不要更改图片文件名
    3. 一致性：确保所有产品文档使用相同的格式标准
    4. 备份：在更新前建议备份原始文件，以防意外

    5. 示例对比

    文件名对比

    • 更新前：QIACHIP KR2202 Instruction Manual AC 110V 220V 433 2ac821bc22e7805eb1f8c51abe323303.md
    • 更新后：QIACHIP_KR2202.md

    格式对比

    标题格式

    • 更新前：# **Product** Size
    • 更新后：## Product Size

    图片引用

    • 更新前：![QIACHIP KR2202 Product Diagram](QIACHIP_KR2202_Product_Diagram.webp)
    • 更新后：![QIACHIP KR2202 Product Diagram](QIACHIP_KR2202_Product_Diagram.webp){ width="50%" .center loading="lazy" }

    组件描述

    • 更新前：简单的列表格式
    • 更新后：两列 HTML 布局

    首页链接更新

    • 更新前：<img src="KR2202/QIACHIP_KR2202_Product_Display_Figure.webp" alt="KR2202 Product Image">
    • 更新后：<img src="KR2202/QIACHIP_KR2202_Product_Diagram.webp" alt="KR2202 Product Image">

    6. 自动化建议

    对于大量文档的批量更新，可以考虑： 1. 创建脚本自动重命名文件 2. 使用正则表达式批量替换格式 3. 使用 Markdown 格式化工具辅助更新

    通过遵循此流程，可以确保所有产品文档保持一致的专业格式，提升用户体验和文档质量。
